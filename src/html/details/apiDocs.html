Notes before moving on

Please take note that the api is accessed by doing http(s) post/get requests. This implies that you can do it in any language, given the language supports get//post requests. If you don't know how to do a get or post request in the language you are working on, I'll provide some links below, but please look it up if I don't have a link for you... It's realtivly simple. Your requests will usually look like "http://hostnameorip/api/command", so if i was hosting on 127.0.0.1, and I wanted to grab my token from the server I would send a post request to http://127.0.0.1/api/login with the parameters username and password. All requests will return data in a JSON format, and will usually start of with {response: {"0":"success/fail", "data":"data"}}. Take our api as a chance to get used to using JSON, because it is one of the best ways to exchange data through an api.

Api

The Current functions we have right now are

/api/login -- This is where login request are sent in the form of a post request. Of course your post request should have the parameters username and password. A post request in python would look like: requests.post("ourserver.com/login/api", data={"username":"pass", "password": "pass"}). You can have two possible returns, a fail and succeed. The fail message usually is {"response" : {"0" : "fail"}}', but a success will return your user token, which will look like this {"response:{"0":"Success", "token":"xxxxx"}"}.

/api/createlogin -- This is where account creation is done, in the form of a post request. The post request have the same parameters as login and returns the same fail output plus the possible "fail" return when there is already an account made with that name. If the account is successfully made, the sever will return a success message along with the key the system has assigned for you. The output would look like {"response:{"0":"Success", "token":"xxxxx"}"}.
