Server Info

Okay so really what is it?

This application is hosted in node.js, built by the head of the JCT Student Team. The core program is based around the module "Express", which can be found via npm. We chose Node.js because it offered a way to control how data comes into our system, and how our system deals with such data. Yes we could accomplish the same with Python or PHP, but we need to strive for modernism. All of the latest companies are pushing for node.js servers or apps, and with facebook pushing react native, keeping it constantly updated, JS is not going to die anytime soon.

So let's talk about node first... What is it? In caveman speak, it's javascript but on your computer, not browser. Such implies that the language now has access to system processes, files etc, etc. It's pretty much Python at this point! In some better terms, people have taken the extreamly powerful Java Script engine created by google chrome and given it access to the system. From node we get things like react-native (Mobile dev in JS), Electron (HTMLXCSSXJS in a desktop app), and of course Express.

Express lets us take http requests from clients and send back data! Of course we could use appache and just host html files, but that is not what I want for this is not a website. Yes you may be looking at a site, but the primary purpose of this server is to get students into the rhythym of dealing with http APIs and JSON objects. Express lets us do both with their pathing, for a user can send a get request to "ourhost/details/our_server" and get this html page, but they can also do a http request to "/api/apiService/q!='data'" and get a JSON file that they can sort through for ideas. That is the idea anyways. That's not just the only reason for creating this, see in order to teach we must first understand the technology. How am I to teach you if I know little of how todo this? This is a learning tool, so we must open it up and explore it!
